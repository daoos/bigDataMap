##  自动生成图表的前端组功能组件库（体验超级傻瓜式的操作，爽）

### 实现的组建的特性

- [ ] 可持久化（将配置文件写入数据库，这样能够便于支持后期的不同角色个性化设置）
- [ ] 使用拖拽的方式作为页面操作的交互逻辑



### 实现步骤

1. 记录需要使用图表的种类。（寻找其中的数据规律）
2. 记录需要经常个性化设置的配置。（抽象其中的配置数据）
3. 选用bootstrap的栅格化作为各个图表容器的布局基础 



### 页面交互作用实现

1. 前期采用传统的表单方式进行定制化。后期采用 拖拽，拉伸的方式进行页面的编辑。


### 技术积累

 1. 寻找一个合适的js操作库 伸缩，拖拽方式（很重要）


### 基本的页面逻辑组成

```
页面的分为三个区域 左边为可选元素，中间为控制区域，右边为控制员的自定义数据

左边相当于：一个个自定义的组建
中间相当于：实时显示系统
右边相当于：个性化的精准化控制

```
